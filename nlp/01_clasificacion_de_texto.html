Procesamiento de Texto üìö

El objetivo principal de esta clase es introducirlos a la clasificaci√≥n de texto en NLP. Para esto, implementaremos varios modelos de clasificaci√≥n destinados a predecir la categor√≠a de noticias de la radio biobio.

Los modelos y m√©todos que usaremos ser√°n los siguientes:

1. Preprocesamiento:

El preprocesamiento es el conjunto de t√©cnicas aplicadas a los datos antes de ser utilizados por un modelo de aprendizaje autom√°tico. Las t√©cnicas que usaremos son:

  • Tokenizaci√≥n: Es el proceso de dividir un texto en unidades m√°s peque√±as llamadas tokens. Los tokens pueden ser palabras, frases o incluso caracteres individuales, seg√∫n el nivel de granularidad que se necesite. Por ejemplo, la frase ‚ÄúHola mundo‚Äù se tokeniza como [‚ÄúHola‚Äù, ‚Äúmundo‚Äù].

  • Stemming: Es una t√©cnica que reduce las palabras a su ra√≠z o forma base. El objetivo es agrupar diferentes formas de una palabra, de modo que puedan tratarse como la misma entidad. Por ejemplo, ‚Äúcorriendo‚Äù, ‚Äúcorre‚Äù y ‚Äúcorrer‚Äù se reducir√≠an a la ra√≠z ‚Äúcorr‚Äù.

  • Lematizaci√≥n: Similar al stemming, pero en lugar de reducir las palabras a su ra√≠z, la lematizaci√≥n convierte las palabras a su forma base o lema seg√∫n el contexto. Por ejemplo, ‚Äúcorrer√°‚Äù se convierte en ‚Äúcorrer‚Äù. La lematizaci√≥n tiene en cuenta el significado gramatical, lo que la hace m√°s precisa que el stemming.

  • Eliminaci√≥n de Stop Words: Las stop words son palabras comunes que no aportan mucho significado a la tarea de an√°lisis, como ‚Äúel‚Äù, ‚Äúla‚Äù, ‚Äúen‚Äù, ‚Äúde‚Äù. Eliminar estas palabras puede ayudar a reducir el ruido en los datos y mejorar la eficiencia del modelo.

2. Bag of Words:

El modelo de Bag of Words (BoW) es una t√©cnica simple de representaci√≥n de texto donde un documento o conjunto de palabras se convierte en un conjunto de palabras √∫nicas y su frecuencia dentro de ese documento. Se ignora la gram√°tica y el orden de las palabras, y solo se cuenta cu√°ntas veces aparece cada palabra en el documento. Es un enfoque com√∫n para convertir texto en datos que pueden ser usados por modelos de aprendizaje autom√°tico. El resultado suele ser una matriz en la que cada fila representa un documento y cada columna representa una palabra √∫nica.

3. Clasificador de Bayes:

El Clasificador de Bayes es un algoritmo de clasificaci√≥n basado en el teorema de Bayes, que calcula la probabilidad de que una instancia pertenezca a una clase dada, en funci√≥n de sus caracter√≠sticas. El modelo de Bayes m√°s conocido es el Naive Bayes, que asume que las caracter√≠sticas son independientes entre s√≠. A pesar de esta suposici√≥n ingenua, Naive Bayes es muy eficaz para tareas de clasificaci√≥n de texto como el filtrado de spam y la categorizaci√≥n de documentos.

4. Logistic Regression (Regresi√≥n Log√≠stica):

La Regresi√≥n Log√≠stica es un modelo de clasificaci√≥n supervisado que se utiliza para predecir la probabilidad de que una instancia pertenezca a una de dos clases posibles. Aunque se llama ‚Äúregresi√≥n‚Äù, se utiliza principalmente para problemas de clasificaci√≥n binaria. La regresi√≥n log√≠stica estima la probabilidad de que una instancia pertenezca a una clase bas√°ndose en una funci√≥n sigmoide que produce un valor entre 0 y 1. Si el valor es mayor que un umbral (generalmente 0.5), se clasifica la instancia en una clase; de lo contrario, en la otra.

¬øQu√© es la clasificaci√≥n de texto?

La clasificaci√≥n de texto consiste en tomar distintos textos y asignarles alguna clase. Dichas clases var√≠an seg√∫n la task que queramos resolver. Por ejemplo:

- Detectar emails SPAM -> SPAM, NO SPAM
- Reviews de Peliculas -> Buena, Mas o menos, Mala, Mal√≠sima, Brutalmente mala.
- An√°lisis de sentimientos de tweets: Felicidad, Tristeza, Enojo, Ira,...

- Categor√≠a de una noticia -> Nacional, Internacional, Econom√≠a, Sociedad, Opini√≥n...
- Autor de un texto -> Cada autor es una clase distinta.

Se define formalmente como:

  • Input:

    • Un documento \(d\)
    • Un conjunto fijo de clases \(c_1, c_2, ..., c_j\)

  • Output:

    • Una clase \(c \in C\) para el documento

Hay dos clases de m√©todos para resolver estos problemas:

  1. Hand-coded Rules ü§ô:

    Establecemos a mano las reglas que permiten detectar las clases.

  2. Supervised Machine Learning üíª:

    Entrenamos clasificadores a partir de muchos ejemplos de documentos etiquetados a mano.

Ahora que tenemos cargado el dataset, podemos implementar nuestro clasificador!

Pipeline

Un pipeline es la definici√≥n de los procesos que llevar√° a cabo el sistema que creemos. Nos permite tener unificados todos los procesos a la vez que simplifica el c√≥digo de nuestro sistema.

En nuestro caso, el pipeline ser√°:

Dataset -> Bag of Words -> NaiveBayes Clf

Bag of Words y CountVectorizer üéí

¬øQu√© era Bag of Words?

Es un modelo en donde transformamos cada una de las oraciones de nuestro dataset en vectores. Cada vector contiene una columna por cada palabra / token del vocabulario. Al procesar el dataset, cada oraci√≥n es mapeada a un vector que cuenta las apariciones de cada una de sus tokens.

Referencia: BoW en wikipedia

Un peque√±o ejemplo

Supongamos que nuestro tokenizador solo separa por espacios.

- Doc1 : 'I love dogs'
- Doc2: 'I hate dogs and knitting.
- Doc3: 'Knitting is my hobby and my passion.

El bag of words quedar√≠a:

CountVectorizer es la clase de scikit que transformar√° nuestro texto a Bag of Words. Fijense que es tremendamente √∫til tenerla dentro de un pipeline ya que fija en un comienzo el vocabulario que tendr√° el Bag of Words, evitando discordancias entre los vectores del conjunto de entrenamiento y el de prueba.

M√©tricas de Evaluaci√≥n

Las m√©tricas definen un puntaje de evaluaci√≥n que indica que tal le fue al sistema. Hay muchas formas distintas de medir su rendimiento. Entre estas, tenemos:

  • precision: El n√∫mero de documentos de una clase clasificados correctamente dividido por el n√∫mero de documentos totales clasificados como esa clase.

  • recall: El n√∫mero de documentos de una clase clasificados correctamente dividido por el n√∫mero de los documentos que se deber√≠an haber clasificado como esa clase.(n√∫mero de documentos reales de esa clase).

  • f1-score : Es la media arm√≥nica entre los anteriores.

  • accuracy : La cantidad de documentos clasificados correctamente versus todos los documentos

Por otra parte, tenemos dos formas de ver dichas m√©tricas agrupadas:

  • Macroaveraging: Se computan las m√©tricas por cada clase y luego de promedia.

  • Microaveraging: Se recolectan las clasificaciones por cada clase, se computa la tabla de contingencia (todos los elementos clasificados) y se evalua. Representa un Macroaveraging ponderado por el n√∫mero de miembros de una clase.

Sistema de clasificaci√≥n con preprocesamiento

Para agregar los tokenizadores en el sistema, creamos funciones que que cada documento de forma individual usando nuestro preprocesador favorito. Luego, CountVectorizer se encargar√° de usar estas funciones sobre todo el dataset.

# Tokenizers para CountVectorizer

# Solo tokenizar el doc usando spacy.
def tokenizer(doc):
    return [x.orth_ for x in nlp(doc)]


# Tokenizar y remover las stopwords del doc
def tokenizer_with_stopwords(doc):
    return [x.orth_ for x in nlp(doc) if x.orth_ not in STOP_WORDS]


# Tokenizar y lematizar.
def tokenizer_with_lemmatization(doc):
    return [x.lemma_ for x in nlp(doc)]

# Tokenizar y hacer stemming.
def tokenizer_with_stemming(doc):
    stemmer = SnowballStemmer('spanish')
    return [stemmer.stem(word) for word in [x.orth_ for x in nlp(doc)]]

Los n-gramas son conjuntos de n-tokens seguidos entre si. La idea de usar esto es que adem√°s, capturemos conceptos.

Por ejemplo, si usamos 2-gramas sobre 'Hoy d√≠a com√≠ lentejas', esta quedar√≠a como:

['hoy dia', 'd√≠a com√≠', 'com√≠ lentejas']

CountVectorizer tiene la opci√≥n para poner n-gramas del tama√±o que tu quieras, y adem√°s incluir mas peque√±os. Todo esto se define en el par√°metro ngram_range.
